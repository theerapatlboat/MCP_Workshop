

**REQUIREMENT**

**Facebook Page  Sales Closing Chatbot**

Chatbot ปิดการขายเชื่อมต่อ Facebook Page — ความต้องการของระบบทั้งหมดที่ต้องพัฒนา

|   |
| :---- |

**Requirement**  —  13 รายการ

| \# | หัวข้อ | รายละเอียด | Requirement |
| :---: | :---- | :---- | :---- |
| **01** | **System Prompt** | ออกแบบ System Prompt ที่กำหนดบุคลิก น้ำเสียง และบทบาทของ Chatbot ให้เป็น "พนักงานขาย" ที่เข้าใจสินค้า พูดจาเป็นมิตร และมีเป้าหมายชัดเจนในการปิดการขาย | ✦ System Prompt ที่พร้อมใช้งาน ระบุบทบาท, ข้อจำกัด, น้ำเสียง และเป้าหมายของ AI |
| **02** | **Context Engineering** | จัดการ Context ที่ส่งเข้า LLM ให้มีประสิทธิภาพ ทั้งข้อมูลลูกค้า ประวัติการสนทนา และข้อมูลสินค้า โดยไม่เกิน Token Limit | ✦ แผนการจัดการ Context: อะไรต้องส่งทุกครั้ง, อะไรส่งตามสถานการณ์, อะไรตัดออกได้ |
| **03** | **RAG (Retrieval-Augmented Generation)** | เชื่อมต่อฐานข้อมูลสินค้า FAQ และโปรโมชัน เพื่อให้ AI ดึงข้อมูลที่ถูกต้องมาตอบลูกค้าแบบ Real-time แทนการท่องจำ | ✦ Knowledge Base ที่จัดหมวดหมู่แล้ว พร้อม Retrieval Pipeline |
| **04** | **Memory** | ออกแบบระบบความจำของ AI ทั้ง Short-term (ภายในบทสนทนา) และ Long-term (จำลูกค้าข้ามบทสนทนา) เพื่อให้บริการแบบ Personalized | ✦ โครงสร้าง Memory: ข้อมูลอะไรที่ต้องจำ, เก็บอย่างไร, เรียกใช้เมื่อไหร่ |
| **05** | **Tool Call / Function Calling** | กำหนด Tools ที่ AI สามารถเรียกใช้ได้ เช่น เช็คสต็อก, คำนวณราคา, สร้างออเดอร์ใหม่, ตรวจสอบสลิปโอนเงิน, สร้างใบเสนอราคา หรือส่ง Link ชำระเงิน เพื่อให้ปิดการขายได้จริง | ✦ รายการ Tools พร้อม Function Schema (ชื่อ, พารามิเตอร์, ผลลัพธ์) |
| **06** | **MCP (Model Context Protocol)** | เชื่อมต่อ Bot กับระบบภายนอกผ่าน MCP เช่น ระบบ CRM, ระบบสต็อก, หรือระบบจัดส่ง เพื่อให้ AI ทำงานร่วมกับ Ecosystem ขององค์กรได้ | ✦ MCP Server ที่เชื่อมต่อกับระบบที่จำเป็น พร้อมรายการ Resources และ Tools |
| **07** | **Guardrail** | ตั้งกฎป้องกันไม่ให้ AI ตอบนอกขอบเขต เช่น ไม่พูดเรื่องการเมือง ไม่ให้ข้อมูลเท็จ ไม่ด่าลูกค้า และไม่ให้ส่วนลดเกินที่กำหนด | ✦ รายการ Guardrails: สิ่งที่ห้ามทำ, การตอบเมื่อเจอคำถามนอกขอบเขต, Fallback Message |
| **08** | **Workflow — Sale Script** | ออกแบบ Flow การขายตั้งแต่ทักทาย → ถามความต้องการ → นำเสนอสินค้า → จัดการ Objection → ปิดการขาย → Follow-up ให้ AI เดินตาม Script อย่างเป็นธรรมชาติ | ✦ Sale Script Workflow Diagram พร้อม Prompt สำหรับแต่ละ Stage |
| **09** | **Multimodal — ข้อความ, รูปภาพ, เสียง** | ออกแบบให้ AI รองรับข้อมูลที่ลูกค้าส่งเข้ามาผ่าน Facebook Messenger ได้ 3 รูปแบบ ได้แก่ ข้อความ, รูปภาพ และเสียง โดย AI ต้องเข้าใจและนำข้อมูลจากทุกรูปแบบไปประมวลผลต่อได้ เช่น ลูกค้าส่งรูปสินค้าที่สนใจ ส่งข้อความถามรายละเอียด หรือส่งเสียงพูดแทนการพิมพ์ | ✦ Flow การรับข้อมูล 3 รูปแบบ (ข้อความ, รูปภาพ, เสียง) และวิธีประมวลผลแต่ละประเภทเพื่อส่งต่อให้ AI ทำงานต่อ |
| **10** | **Data Structure — จัดการข้อมูลสินค้า** | วางโครงสร้างการจัดเก็บข้อมูลสินค้าและรูปภาพสินค้า กำหนดว่าจะเก็บที่ไหน (Database, Object Storage, Vector DB) จัดหมวดหมู่อย่างไร และเชื่อมโยงข้อมูลรายละเอียดสินค้ากับรูปภาพอย่างไร เพื่อให้ AI ค้นหาและตอบลูกค้าได้ถูกต้อง (ข้อความและเสียงจากลูกค้าไม่ต้องจัดเก็บ) | ✦ Data Schema และ Storage Architecture สำหรับข้อมูลสินค้า (รายละเอียดสินค้า \+ รูปภาพ) |
| **11** | **Evaluation & Testing** | ออกแบบวิธีวัดผลว่า AI ตอบดีหรือไม่ดี สร้าง Test Case สำหรับทดสอบสถานการณ์ต่าง ๆ เช่น ลูกค้าถามราคา, ลูกค้าต่อราคา, ลูกค้าถามนอกเรื่อง วัดค่า Accuracy, Hallucination Rate และ Response Time เพื่อปรับปรุงได้อย่างมีหลักการ | ✦ ชุด Test Case พร้อมเกณฑ์การให้คะแนน และวิธีวัดผลที่วัดซ้ำได้ |
| **12** | **Error Handling & Fallback** | ออกแบบการจัดการเมื่อ AI ไม่เข้าใจคำถาม ตอบไม่ได้ หรือระบบล่ม กำหนดว่าเมื่อไหร่ต้องส่งต่อให้คนจริง (Human Handoff) Retry Logic เป็นอย่างไร และข้อความ Fallback ที่ลูกค้าจะได้รับในแต่ละกรณี | ✦ Error Handling Flow: เงื่อนไขการ Fallback, ข้อความแจ้งลูกค้า และจุดส่งต่อให้คนจริง |
| **13** | **Prompt Versioning & Iteration** | จัดการ version ของ Prompt เมื่อต้องปรับปรุง ทำให้ track ได้ว่า version ไหนทำงานดีกว่า เปรียบเทียบผลลัพธ์ระหว่าง version เก่ากับใหม่ และสามารถ rollback ได้เมื่อ version ใหม่มีปัญหา | ✦ ระบบจัดการ Prompt Version: วิธี naming, การเก็บ changelog และวิธี A/B test เปรียบเทียบ version |

**ภาพรวมของระบบ**

ระบบ Chatbot ปิดการขายที่เชื่อมต่อ Facebook Page ครอบคลุมตั้งแต่ System Prompt, Context Engineering, RAG, Memory, Tool Call, MCP, Guardrail, Sale Script Workflow, Multimodal, Data Structure, Evaluation & Testing, Error Handling & Fallback ไปจนถึง Prompt Versioning & Iteration
